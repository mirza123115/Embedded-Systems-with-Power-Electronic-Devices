12.1  Introduction
1.  (1)  What is a Sensor?
It is an "agent" that detects the environmental parameters like pressure, temperature, altitude, humidity, weight, color, etc. and produces proportional electrical signal. For example: LM35 is a temperature sensor, and it produces an electrical voltage of 250 mV when the environment temperature is 250C. The sensor produces 300 mV signal when the environmental temperature is 300C.

(2)  Sensor Accuracy

(3)  Sensor Precision

2.  Commonly used Sensors in Household Appliances and Industrial Machinery.
(1)  LM35: Analog type Temperature Sensor

Figure-12.1: LM35 temperature sensor with UNO

(a)  Response Equation of LM35 (found using A(250 mV, 250c) and B(300 mV, 300C points of Section-1(1))
T0C ~= 100*VDT   //VDT=DC Voltage at sensor output pin when temperature is T0C temp.
==> T0C = 100*(VREF/1023)*analogRead(A4);

(2)  DS18B20: Digital type Temperature Sensor

Figure-12.2: DS18B20  temperature sensor with UNO

(a)  Response Profile
Temperature changes gradually and hence there is a gradual change at the output of the "Sensing Element" of DS18B20 sensor.

The sensor's electronics is designed in such a way such that a change of 0.06250C temperature in the environment, the BIT 0 of the following "Temperature Register" (Fig-12.3) assumes HIGH (1) state.

Read the bits of this register and multiply by the respective positional weights, you get the calibrated temperature value.

Figure-12.3: Bit layout of the temperature register of DS18B20 sensor

(3)  I2C Bus Driven 5V BME280 Sensor Breakout Board (with on-board level sifters and Voltage Regulator): Pressure, Temperature, and Humidity Sensor.

A "Breakout Board" contains facilities to mount pins/sockets for making jumper connections with Arduino.


Figure-12.4: BME280 sensor is on a breakout board

(4)  I2C Bus Driven 5V MPU6050 Sensor Breakout Board: 3-axis Accelerometer and 3-axis Gyroscope.
It is a Micro Electro-Mechanical Systems (MEMS). The sensor measures acceleration, velocity, orientation, displacement, and many other motion related parameters of a system or object.

Figure-12.5:

(5)  Load cell (30 kg)
It is a "wheatstone bridge" type sensor. It produces a DC output voltage proportional to weight placed on it. The load cell of Fig-12.6 produces around 10 mV signal when 30 kg weight/load is placed on it.


Figure-12.6:  Wheatstone bridge type load cell

3.  Calibration Philosophy of a Load Cell Sensor
(1)  An "ideal load cell sensor" (say: Fig-6) will produce "zero output voltage" when "no weight' is placed on it. It will also output 5 mV signal when 15 kg load is placed on it. The response curve of an ideal sensor passes through origin and make 45o angle with the abscissa (Fig-12.7 drawn not to scale).


Figure-12.7: Response curve of an ideal load cell

(2)  An ideal load cell exits in the ideal mind/world. In the practical field, there is a "real load cell sensor" which produces some amount of finite output voltage when there is no load placed on it. This voltage is known as "offset". The real load cell also produces more output voltage (say 5.3 mV) than what should be (5 mV) when 15 kg weight/load is applied on it. It demonstrates that the real load cell is amplifying the signal or to say in other way that the load cell has applied "gain" to the signal. Fig--12.8 (drawn not to scale) depicts the response curve of a real sensor.  


Figure-12.8: Response curve of a real load cell

(3)  Now, we see that a real load cell (in general sensor) has deviated from its ideal nature due to some imperfections that has introduced into it during manufacturing. Every real sensor has "gain" and "offset".

(4)  If we know the value of "offset", the we can subtract it from the actual/measured measurement. If we know the value of "gain", then we can reduce the actual/measurement value by multiplying it with the inverse of the gain (the gain factor).

(5)  Calibration is the task of finding the "gain" and "offset" of the real sensor from the 'Real Response" curve of the load cell (in general the sensor).

12.3  Programming of I2C Bus driven BME280 Sensor using 5VBME280 Beakout Board of Fig-12.4.
In this Section, We will develop a sketch to read temperature data from BME280 sensor and then the value will be shown on Serial Monitor. Pressure and humidity measurement and display will be done in some other sections.

The BME280 Sensor can be operated using SPI Port or I2C Bus. We will use I2C Bus which is directly supported by the 5VBME280 Breakout Board of Fig-12.4.

1.  (1)  Let us check that the sensor placed on the breakout board of Fig-12.4 is really a BME280 sensor by reading the "ID code (0x60)" from the sensor's "ID Register". You may ask question why is it not enough to know the I2C Bus address?

(2)  Look at Fig-12.10 and check that the "internal address" of the "ID Register" is 0xD0 and the "ID Code" it holds is 0x60.

(3)  The codes to be executed are:
Code: [Select]
Wire.beginTransmission(0x76);  //addressing the "Whole Sensor" at I2C Bus address x76
Wire.write(0XD0);                              //pointing at the ID Register
Wire.endTransmission();
//-----------------------
Wire.requestFrom(0x76, 1);              //request to read 1-byte data from IN Register pointed in Step-c
byte idValue = Wire.read();               //idValue should hold 0x60
//-------------------------
if(idValue == 0x60)
{
     Serial.println("Sensor's ID found as 0x60.");
}
else
{
     Serial.println("Sensor's ID is not found...!");
     while(1):    //wait for ever
}
 
2.  (1)  Set "Normal Mode" operation of the sensor by putting 1/1 into bit1/bit0 positions of the ctrl_meas register (Fig-12.10). The address of the ctrl_meas register is given as: 0xF4.  

(2)  Write codes for the task of Step-2(1).
Code: [Select]
Wire.beginTransmission(0x76);  //addressing the "Whole Sensor" at I2C Bus address x76
Wire.write(0XF4);                              //pointing at ctrl_meas Register
Wire.write(0x03);                              //data for bit1/bit0 of the ctrl_meas register.
Wire.endTransmission();
//-----------------------
Note:  While writing into bit1/bit0 of ctrl_meas register, the other bits should remain undisturbed. But, the codes of Step-2(2) corrupt bit7-bit2 by putting 0s there. Therefore, the codes of Step-2(2) are to be modified as follows so that only bit1/bit0 bits are addressed.
Code: [Select]
[code]Wire.beginTransmission(0x76);   //the sensor holds the register
Wire.write(0xF4);   //point the ctrl_meas register
Wire.endTransmission();
//------------------------------------------------------
Wire.requestFrom(0x76, 1);    //read the content of ctrl_meas register
byte x = Wire.read();
x = x | 0b00000011;             //only bi1/bit0 of ctrl_meas are changed; other bits remain unrouched.
//-----------------------------------------------------
Wire.beginTransmission(0x76);
Wire.write(0xF4);
Wire.write(x);
Wire.endTransmission();
3.  (1)  Now, we are going to read temperature data from the "Temperature Register" of BME280 sensor. In Fig-12.10, we observe that the Temperature Register is composed of the following 3 locations (Fig-12.18):

Figure-12.18: Bit pattern of Temperature Register.

(2)  Fig-12.18 says that the Temperature Register has 20 active bits and they are formed as:

Figure-12.19: Formation of 20-bit active bits for Temperature Register

Code: [Select]
long tempReg = (long)|(long)(temp_msb<<12)|(long)(temp_lsb<<4)|(long)(temp_xlsb>>4)
(3)  In Normal Mode operation, the signal conversion (from varying DC to digital form) within BME280 takes place perpetually; there is no need to issue any command from the host (the Arduino UNO) computer. As long as the conversion is in progress, the bit3 (measuring) of Status Register (fig-12.10) remains at HIGH state. After the conversion, the bit3 assumes LOW state and the data from ADC is transferred into the registers of Fig-12.18. Now, our task will be to poll the measuring bit to see when it goes to LOW state and then bring those data to UNO. The codes are:
Code: [Select]
Wire.beginTransmission(0x76);   //the sensor holds the register
Wire.write(0xF3);   //point the status register; Fig-12.10
Wire.endTransmission();
//------------------------------------------------------
do
{
   Wire.requestFrom(0x76, 1);    //read the content of status register pointed above
   byte x = Wire.read();
}
while(bitRead(x, 3) != LOW);     //keep checking until conversion is done
//--------------------------------------------------------------------------------

4.  Read Temperature Data by executing the following codes:
Code: [Select]
Wire.beginTransmission(0x76);
Wire.write(0xFA);                       //pointing temp_msb register; auto increment
Wire.endTransmission();
//-------------------------------
byte m = Wire.requestFrom(0x76, 3);    //request to read 3-byte data
uint8_t temp_msb = Wire.read();            //MSByte
uint8_t temp_lsb = Wire.read();              //NSByte
uint8_t temp_xlsb = Wire.read();            //LSByte

5.  Manipulate the contents of the variables of Step-4 as per Fig-12.20 so that the resultant 20 active bits of temperature data are saved in the variable tempReg.

Figure-12.20: Formation of 20-bit active bits for Temperature Register

Code: [Select]
uint32_t tempReg = (uint32_t)(temp_msb<<12)|(uint32_t)(temp_lsb<<4)|(uint32_t)(temp_xlsb>>4)
6.  Let us process the value of tempReg variable of Step-4 to get the actual value of temperature in oC with 2-digit precision after the decimal point.

(3)  Select "Normal Mode of Operation"
bme1.parameter.sensorMode = 0b11;

(4)  Set "factor" of IIR (Infinite Impulse Response) filter. A high value meas less noise
bme1.parameter.IIRfilter = 0b100;     //factor 16

(5)  Set "over sampling factor" for humidity sensor. Higher value means less noise.
bme1.parameter.humidOversampling = 0b101;    //factor 16

(6)  Set "over sampling factor" for temperature sensor. Higher value means less noise.
bme1.parameter.tempOversampling = 0b101;    //factor 16

(7)  Set "over sampling factor" for pressure sensor. Higher value means less noise.
bme1.parameter.pressOversampling = 0b101;    //factor 16

(8)  Put present pressure corrected at sea level
bme1.parameter.pressureSeaLevel = 1013.25;    //default 1013.25 hPa = hecta Pascal

(9)  Put current average temperature outside
bme1.parameter.tempOutsideCelsius = 15;     //default 15 degree centrigrade

(10)  heck if the Sensor is present by reading its ID (0x60)
Code: [Select]

  if (bme1.init() != 0x60)

Sketch:
Code: [Select]
#include <Wire.h>
#include "BlueDot_BME280.h"

BlueDot_BME280 bme;
int bmeDetected = 0;    //Checks if Sensor 1 is available

void setup()
{
  Serial.begin(9600);
  Serial.println(F("Basic Weather Station"));
  
  bme.parameter.communication = 0;                    //I2C communication for Sensor  (bme1)
  bme.parameter.I2CAddress = 0x76;                    //I2C Address for Sensor  (bme1)
  bme.parameter.sensorMode = 0b11;                    //Setup Sensor mode for Sensor
  bme.parameter.IIRfilter = 0b100;                   //IIR Filter for Sensor
  bme.parameter.humidOversampling = 0b101;            //Humidity Oversampling for Sensor
  bme.parameter.tempOversampling = 0b101;              //Temperature Oversampling for Sensor
  bme.parameter.pressOversampling = 0b101;             //Pressure Oversampling for Sensor
  bme.parameter.pressureSeaLevel = 1013.25;            //default value of 1013.25 hPa (Sensor)
  bme.parameter.tempOutsideCelsius = 15;               //default value of 15C

  if (bme.init() != 0x60)
  {
    Serial.println(F("Ops! First BME280 Sensor not found!"));
    bmeDetected = 0;
    while (1); //wait for ever
  }
  else
  {
    Serial.println(F("First BME280 Sensor detected!"));
    bmeDetected = 1;
  }

  Serial.println();
  Serial.println();
}

void loop()
{
  Serial.print("Temperature Sensor [C]:  ");
  float bmeTemp = bme.readTempC();
  Serial.println(bmeTemp, 2);
  //------------------------------------------
  Serial.print("Humidity Sensor [%]:     ");
  float bmeHumid = bme.readHumidity();
  Serial.println(bmeHumid, 2);
  //-------------------------------------------
  Serial.print("Pressure Sensor [hPa]:  ");
  float bmePress = bme.readPressure();
  Serial.println(bmePress, 2);
  //--------------------------------------------
  Serial.print("Altitude Sensor [m]:      ");
  float bmeAlt = bme.readAltitudeMeter();
  Serial.println(bmeAlt, 2);
  //----------------------------------------------
  Serial.println("****************************************");
  Serial.println();
  
  delay(3000);  //test interval
}
